## Decisions made
- **VM name `iksar_buddy`** (opsec В§7.5).
- **8 GB RAM** вЂ” owner overrode my 6 GB rec after seeing host had ~9 GB free.
  Added virtio balloon so host can reclaim under pressure. Watch swap on boot.
- **OS disk on SATA, not virtio-blk** вЂ” avoids injecting a storage driver into
  WinPE; perf irrelevant for a 2004 client. NIC stays virtio.
- **virtio NIC dead during OOBE (no in-box Win10 driver) = forces offline
  install** вЂ” exactly what we want. Bring net up via NetKVM post-install.
- **Win10 22H2 (19045) via uupdump**, not mido вЂ” mido's MS consumer-ISO flow is
  dead (Win10 EOL Oct 2025). Win10 EOL is a *plus* here: no feature updates to
  shift the UI under the pixel sensors (PROJECT.md wants this frozen + snapshotted).
- **Built ISO with xorriso (no UDF)** вЂ” xorriso 1.5.8 lacks UDF; ISO level 3
  multi-extent handles the >4 GB install.wim, UEFI El Torito via efisys.bin.
  Patched uupdump `convert.sh` mux command accordingly (`convert.sh.orig` kept).

## Host gotchas hit (all worked around without touching the live stack)
- Remote login shell is **fish** вЂ” run remote scripts via `bash -s`.
- `aria2` (uupdump dep) needs `libnettle.so.9`; host is on nettle 3.10.2 (so.8).
  Upgrading nettle would break `dnsmasq` + `wget` (host DNS!) вЂ” so did NOT upgrade.
  Extracted `libnettle.so.9`/`libhogweed.so.7` from the nettle-4.0 pkg into
  `~/ib-build/lib` and ran aria2 with `LD_LIBRARY_PATH`. Fully contained.
- `mkisofs`/`genisoimage` absent on Arch; created `/usr/local/bin/mkisofs` +
  `genisoimage` shims в†’ `xorriso -as mkisofs` (rewrites `--opt`в†’`-opt`).
