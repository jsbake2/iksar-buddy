## Root cause (the important finding)
Craft bot 2 (iksar_buddy3) was NOT an OCR or recipe-match failure. The **Windows
taskbar was drawn over the bottom of EQ2** (EQ2 windowed-fullscreen lost foreground —
likely a Windows notification / the search box grabbed focus). The chat-safety region
is calibrated at `y:1019` (the EQ2 chat input line). With the taskbar up, that region
sat on the bright Windows search box / icons.
- Live sample: `chat_input` thresholded brightness = **208** vs fail threshold **25**.
- So `chat_safe()` returned False for every recipe → `_select_recipe` aborted the type
  (`recipe type ABORTED (chat unsafe / not in-world)`) → 6 recipes done=0 → writ
  "incomplete." The fail-safe working as designed; the screen state was wrong.
- iksar_buddy2 was fine because it's true fullscreen (no taskbar).
Owner restored focus; re-sampled brightness = **0**, game_present blue = 428 → clear.

(Separately, iksar_buddy2's earlier "in-guest list ended at 1/2" is the known in-guest
start-confirm gap on continuation combines — NOT addressed this session.)

## Changed
- `forge/worker.py` — new `_wait_chat_safe()`: on an unsafe gate, set state `blocked`,
  log/emit "BLOCKED: chat-safety failing — Restore fullscreen+focus", and WAIT
  (re-check every 1s) until safe (or job stopped), then resume the same writ with
  done-counts intact.
- `forge/worker.py` `_select_recipe` — replaced the silent chat-unsafe abort+skip with
  a call to `_wait_chat_safe()` then fall through and type. A buried EQ2 can no longer
  march through a whole writ doing nothing and report "incomplete."

## Decisions made
- Don't bound the BLOCKED wait — wait indefinitely until safe or stopped. Burning a
  writ silently is worse than pausing; the owner sees BLOCKED and fixes focus, progress
  preserved. Owner can always Stop.
- Pre-flight guard lives inside `_select_recipe` (covers first recipe AND mid-writ
  taskbar pops) rather than only the writ loop top.
- Host-side only change (forge/worker.py). Nothing to push into any guest, so
  iksar_buddy2 was NOT started/touched (owner: may start to push code but DO NOT log in
  the account — in use elsewhere).
