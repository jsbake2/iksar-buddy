## Root causes found + fixed (18 commits, see git log)
- Reaction detection NEVER fired for two compounding reasons: **opencv wasn't installed**
  in the deployed venv (matching silently no-op'd), and the **confidence threshold was
  0.80** but real counters match **0.58–0.70** (measured live; bg 0.14). Set 0.45.
- Reaction templates (42×42) were **bigger than the 40×40 watch region** → matchTemplate
  skipped all 3 → None. Templates 40×40, watch region **48×48** (bg 0.15, counter 0.58–0.70).
- Counter art was pressed via **ibkey (async scheduled task)** → landed late / DROPPED on
  collision → counters failed. Switched art presses (1–6) to **virsh send-key**
  (synchronous, instant). Counters now land.
- **Debounce**: was mashing each event 5–7×; now one press per NEW counter, then resume
  filler spam until it clears (owner's spec).
- **Recipe load needs a DOUBLE-click** (single highlights only); added `Guest.double_click`
  (Event-mode AHK `Click("x y 2")`). Match returns the icon-x at the OCR'd row-Y (rows shift).
- **Create-then-Begin**: the start button isn't a fixed gold Begin; double-click loads,
  **Create** opens the setup where Begin@784,707 appears (even from a polluted state).
- **game_present** (screenshot) now gates Start instead of flaky `eq2_running` (guest-exec
  intermittently returns empty → false "EQ2 not running" bail).
- Search-focus race, clear-X, single clean search w/ poll, wait-for-click-to-land,
  Start-supersedes-running-job, single-mode recipe falls back to saved (placeholder).
