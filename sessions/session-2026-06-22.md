## Session 2 (afternoon/evening 2026-06-22) — spawn list cracked, harvest detection

### Dropped Cheat Engine entirely
CE-over-SPICE was unworkable (no keyboard remotely, game popping foreground, autorun never
firing). Built a **headless hardware-breakpoint "find what accesses"** instead:
`harvest/find_accessors.py` runs in-guest via guest-exec, DebugActiveProcess + DR0/DR7
watchpoint on every thread, captures writer regs + a stack-walk. Fully scriptable, no GUI.
- **CRASH BUG fixed:** DebugActiveProcessStop does NOT clear DR regs — leftover watchpoints
  fire #DB with no debugger → EQ2 crashes seconds after each capture. Now disarms all
  threads before detach. (This was killing the game twice; relogged via /api/char each time.)

### SOLVED
- **Heading** = `[EverQuest2.exe+0x1822b74]` (4th float after XYZ; turn-and-diff confirmed).
  Live on dashboard (compass — N/E/S/W mapping still to calibrate).
- **Spawn list (mobs/NPCs/players)** = vtable `EverQuest2.exe+0x1782848`, world pos at
  obj+0x20. ~86 actors enumerated with real coords. VERIFIED: player object traced to
  "Furyflatulence" and followed the player on move. Name path `[+0x200]->[+0xb8]->+0x20`.
- **Harvest detection (the keystone)** = EQ2 log line, confirmed live mining Stonecrest Ore:
  `You mine 1 \aITEM ..:carbonite cluster\/a from the stonecrest ore.` Verb→type
  (mine/forage/gather/fell/trap), item, qty, node name. Node holds ~3 sets, despawns when
  depleted. Node typing comes from the LOG, not memory. See [[harvest-log-signal]].
- **Logging rule** [[verify-logging-on-bot-start]]: /log defaults OFF each relog; Fury's log
  was frozen 06:22 while clock was 18:09. Every bot must self-verify logging on start.
