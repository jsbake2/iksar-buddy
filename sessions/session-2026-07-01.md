## Decisions made
- Toast transport = a dedicated `snapshot.notice` field (not diffing the noisy event stream) — explicit, one field, client fires once on new ts. Mirrors forge's existing `failure_report` ts-diff contract.
- Timed-writ bug root cause: `_reread_writ_queue` returns `[]` on BOTH a genuinely-clear journal AND a failed/blank OCR read; the loop treated `[]` as "complete", so a single OCR blip aborted retries and falsely reported done. Fix = require `timed_writ_confirm_empties` (default 2) CONSECUTIVE empty reads before calling it clear; one blip just re-checks. Chose consecutive-empties over refactoring `ocr_journal` to surface raw text (localized, low-risk).
- OS notifications need a user gesture for permission — toast.js requests it on first click.
- Own-death gated on `game_present` to avoid false "Bot has died" when not in-world.
