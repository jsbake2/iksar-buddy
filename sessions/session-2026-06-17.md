## Pink #3 counter — root cause was press-gating, not color tuning — commits b7..,..
**Symptom:** sage/scribe crafts died at ~33% durability when counter #3 (pink icon w/ red
accents) appeared. **Real bug (not the color thresholds):** the press loop in
craft_reflex.run() was gated on `self._cnt_resolved is None`. The green/red tint resolver
false-resolves 'red' on the pink icon's OWN onset frame (debug: dr=24 dg=-25), so the
mash stopped after a single key-3 press — the event never cleared, durability bled out.
Fix: decouple. Mash the matched art the ENTIRE time the icon is visible (icon vanishing =
event over). green/red resolution kept as dashboard STATS only, no longer gates pressing.
This also makes #1/#2 more robust and removes any dependence on reading the pink tint, so
the watch-crop images (which a bail craft kept wiping) are no longer needed to fix it.

**Guest drift fix:** guests run a persistent C:\ib\agent\craft_reflex.py that was only
updated by hand → they silently ran stale logic. Added Guest.push_file + Guest.sync_reflex;
controller._launch now pushes the canonical guest_agent/craft_reflex.py + bounces ibagent
on every successful launch. So buddy/buddy2 (powered off now) self-update next launch.
