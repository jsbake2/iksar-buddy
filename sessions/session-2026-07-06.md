## Decisions made

- LoginDriver moved to shared/ NOW (REFACTOR listed it as "consider") — brain importing
  from forge was the worst layering violation and the move was mechanical.
- forge/guest.py + forge/login.py kept as import shims for one release, then delete.
- infra/vm/host-helpers/gexec.py left as-is: it's a deliberately standalone scp-able CLI
  helper; porting it onto shared.guest would break its zero-dependency property.
- re_tools import shared.guest directly (no sys.path hack) — they're run from the repo
  checkout on 10.0.0.16 anyway; editable install is the supported way.
- exec_out returns "" on failure (matching the old `_gx`), agent_cmd raises — the
  distinction is deliberate: fire-and-read tools want soft-fail, inject wants loud-fail.
