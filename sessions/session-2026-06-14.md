## Decisions made
- **Direct login over char-select.** char-select OCR was unreliable (changing list
  order, duplicate names across servers, click offset). Logging straight into the
  named character via the EQ2 login form sidesteps all of it. Owner agreed.
- **`SetKeyDelay 55, 45` in Event-mode `{Raw}`** — Event mode raced the Shift key and
  corrupted password symbols (`3`→`#`, `$`→`4`, still 13 chars so the dot-count looked
  right). The delay lets Shift settle. THE key fix; without it login is rejected.
- **EQ2 dialog/button clicks must use `Guest.click` (ibgclick)**, never AHK `Click()`
  (EQ2 ignores it). First-login "Import UI Settings" dialog dismisses with `{Enter}`.
- **Host-orchestrated, not one monolithic AHK** — detection/gating in Python where it's
  debuggable; each guest action a tiny AHK. LaunchPad PLAY button is a stable fixed
  pixel (owner-confirmed); detect orange face at (1300,752).
- **Disk grow:** live `virsh blockresize` + online `Resize-Partition` for the running
  VM; `qemu-img resize +20G` + boot/extend/shutdown for the off ones. C: is the last
  partition on all three (no trailing recovery) so it extends cleanly.
