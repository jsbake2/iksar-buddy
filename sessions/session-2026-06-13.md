## CPU smoothness fix (owner: "hammers the cpus hard")
Root cause: GPU-less crafter VM software-renders EQ2 (3D) on the CPU; uncapped FPS
pegs its vCPUs. Healer offloads to the 4070 so it's light.
- **Hard cap (applied live+config + baked into iksar_buddy2.xml):** cputune
  global_quota=200000 / period=100000 = crafter limited to ~2 cores, can't hammer
  the host no matter what. Carries to the iksar_buddy3 clone.
- **Real cure (owner, in-game):** EQ2 Options → Display → Max Frame Rate ~24,
  **Max Frame Rate (Background) ~5** (bot runs headless → VM almost always background
  → ~5 FPS → near-idle CPU), lowest graphics preset. Clones forward to crafter #2.
- Optional later: drop crafter vCPUs 6→4 (with the quota, 6 just adds vcpu threads);
  set on the iksar_buddy3 clone.

## CPU cap REVERTED — it strangled the client
The 2-core quota made the crafter unusable (owner: "can't even open options menu").
GPU-less software render + zone loading needs real cores. Fix:
- Removed the global_quota cap (live+config) — full 6 vCPUs again.
- Widened vcpu pin from 1:1 (cpu8-13) to float across cpu8-15 (4 free physical
  P-cores, no HT contention; healer owns cpu0-7). Crafter now ~5.5 cores when busy,
  responsive. host load ~6/24.
- XML updated to match (no hard cap; wide pin). Clones inherit this.
- The ONLY CPU lever now is the in-game FPS cap (Max ~24 / Background ~5) — cuts CPU
  at the source without lag. No hard quota.
