## Decisions made
- **The whole execution path already existed** — the keymap already renders `rez`
  as a mappable ability (Ctrl+= live / Ctrl+1 repo), and the agent already does
  target-then-cast (presses the slot's F-key, then the ability key) for any
  COMMAND with a `target_slot`. `rez_priority`, `REZ_LOOP`, `FORCE_REZ` were all
  defined but **vestigial** — nothing in `decide()` ever emitted a rez. So the
  only real gap was the decision. Implemented that; reused `rez_priority`.
- **Live-emergency guard, not combat-gated.** Owner said "rez immediately," so rez
  fires in ANY state (combat / wipe-recovery / OOC) — EXCEPT while a living member
  is below the emergency HP line (a locked 6s rez over a dying teammate wipes the
  group; heal them first, rez slots into the next gap). Also skips mid-cast so it
  doesn't interrupt an in-flight heal.
- **Did NOT touch live profile keybinds.** Live fury (`~/ib-data/config/profiles/
  furyflatulence.yaml`) has `rez: Ctrl+=` (repo says Ctrl+1) — classic profile
  drift; auto-rez uses whatever key is live, so no need to overwrite. Only the
  additive `auto_rez` knob was injected into live thresholds.

- **Dirge heal is a DISTINCT role (`st_heal`), not `direct_heal`/`heal`.** Two
  reasons: (1) `heal`/`direct_heal` are in `_MEMBER_ACTIONS`/the healer `decide()`
  stack — reusing them would double-fire the dirge heal through the healer priority
  loop at the healer's aggressive 0.90 threshold; (2) a separate role gets its own
  `dirge_heal_hp` threshold so a support toon tops off only when meaningfully hurt
  (0.70) instead of proactively at 0.90. The heartbeat spec is role-gated (empty
  `key_for('st_heal')` on healer profiles → no-op), so it's dirge-only automatically.
