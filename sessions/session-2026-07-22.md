# Session Log — 2026-07-22

## Scope this session
Add a resurrect capability: the bot must detect a group member has died and rez
them immediately (and make sure rez is a mappable keymap button).

## Changed
- `brain/policy.py` — NEW auto-rez block in `decide()`. When a present member is
  `dead` and no LIVING member is in emergency and we aren't mid-cast, emit a `rez`
  action targeting the downed member. Target order = `rez_priority` (role list,
  tank-first tiebreak). Placed right after the emergency-heal block and BEFORE the
  cures/heals block, so rez ranks near the top of the priority burst (outranks
  routine heals/cures) but still yields to a living emergency. Gated on
  `auto_rez` threshold (default True).
- `brain/server.py` — added `"rez": 10.0` to `ACTION_COOLDOWN_S`. Rez is a ~6s
  locked cast + accept latency; the cooldown stops us stutter-recasting our own
  rez every sensor tick. Tunable via `thresholds.cooldowns_s.rez`.
- `config/thresholds.yaml` — added `auto_rez: true` and `cooldowns_s.rez: 10.0`.
- `config/profiles/{croolst,croalst,furyflatulence,jenskin}.yaml` — flipped `rez`
  `mode: manual → auto` + refreshed desc (it's auto-cast now). Cosmetic only:
  `mode` is a keymap-UI hint, not consumed by the policy.
- `tests/test_policy.py` — 7 new tests (fires on death, tank-first priority,
  living-emergency blocks rez, outranks routine heal, fires OOC, disable flag,
  no-rez-when-nobody-dead). Full suite 113 passed.

## Decisions made
- **The whole execution path already existed** — the keymap already renders `rez`
  as a mappable ability (Ctrl+= live / Ctrl+1 repo), and the agent already does
  target-then-cast (presses the slot's F-key, then the ability key) for any
  COMMAND with a `target_slot`. `rez_priority`, `REZ_LOOP`, `FORCE_REZ` were all
  defined but **vestigial** — nothing in `decide()` ever emitted a rez. So the
  only real gap was the decision. Implemented that; reused `rez_priority`.
- **Live-emergency guard, not combat-gated.** Owner said "rez immediately," so rez
  fires in ANY state (combat / wipe-recovery / OOC) — EXCEPT while a living member
  is below the emergency HP line (a locked 6s rez over a dying teammate wipes the
  group; heal them first, rez slots into the next gap). Also skips mid-cast so it
  doesn't interrupt an in-flight heal.
- **Did NOT touch live profile keybinds.** Live fury (`~/ib-data/config/profiles/
  furyflatulence.yaml`) has `rez: Ctrl+=` (repo says Ctrl+1) — classic profile
  drift; auto-rez uses whatever key is live, so no need to overwrite. Only the
  additive `auto_rez` knob was injected into live thresholds.
